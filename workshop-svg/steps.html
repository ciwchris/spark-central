Using SVG to draw images

Drawing area

All images need to be drawn within an area. A svg element is used to make this area. The area is created with a width and a height. The width and height are attributes of the svg element.

<svg width="100" height="50"> </svg>

When an item is placed in the drawing area its location needs to be set. The direction to the right is specified with one value. This value is represented by 'x'. The direction down is another value, represented by 'y'. In the example below the width of the drawing area is 100 and the height 50. Therefore valid values for x can be anywhere between 0 and 100 and valid y values between 0 and 50.

A point on the drawing area is represented by a point (x, y). The example below shows the point (10, 15).

A line can be created between two points. The line starts at the point (10, 15) and ends at the point (75, 40).

Lines

Draw lines from the first point (x1, y1) to the second point (x2, y2)

  • Set the color of the line: stroke="green"
  • Try different color names
<svg width="400" height="50"> <line x1="50" y1="10" x2="350" y2="40" stroke="green" /> </svg>

Try including these other attributes with lines:

  • Line size: stroke-width="10"
  • How a line starts and ends (values: round and square): stroke-linecap="round"
  • Try drawing multiple lines
<svg width="400" height="50"> <line x1="50" y1="30" x2="350" y2="20" stroke="slateblue" stroke-width="10" stroke-linecap="round" /> <line x1="240" y1="15" x2="180" y2="35" stroke="paleturquoise" stroke-width="20" stroke-linecap="square" /> </svg>

Rectangles

Draw rectangles from upper left point (x, y) with a width and height

<svg width="400" height="50"> <rect x="5" y="5" width="390" height="40" /> </svg>

Try including these other attributes with rectangles:

  • Round corners: rx="5" ry="5"
  • Color: fill="dimgray"
  • Border color and size: stroke="orchid" stroke-width="5"
  • Transparent: fill="transparent"
  • Make it see through (values: 0.0-1.0): fill-opacity=".5"
  • Also make borders see through (values: 0.0-1.0): opacity=".5"
<svg width="400" height="50"> <rect x="5" y="5" width="80" height="30" fill="orangered" rx="10" ry="10" /> <rect x="110" y="2" width="80" height="45" fill="transparent" stroke="orchid" stroke-width="5" /> <rect x="250" y="5" width="100" height="35" rx="20" ry="20" fill="seagreen" stroke="lightseagreen" stroke-width="5" /> <rect x="220" y="20" width="60" height="30" fill="powderblue" fill-opacity=".5" stroke="royalblue" stroke-width="5" /> <rect x="320" y="20" width="60" height="30" fill="powderblue" opacity=".5" stroke="royalblue" stroke-width="5" /> </svg>

Circles & Ellipses

Draw circles at a point (cx, cy) with a radius (the size of the circle).

<svg width="400" height="50"> <circle cx="200" cy="25" r="20" fill="yellow" /> </svg>

Draw ellipses at a point (cx, cy) with a both a vertical and horizontal radius (rx, ry)

<svg width="400" height="50"> <ellipse cx="200" cy="25" rx="50" ry="20" fill="greenyellow" /> </svg>

Try using the other attributes you learned with rectangles

<svg width="400" height="100"> <circle cx="200" cy="50" r="40" fill="transparent" stroke="lavender" stroke-width="3"/> <ellipse cx="200" cy="50" rx="40" ry="15" fill="transparent" stroke="lavender" stroke-width="3"/> </svg>

Text

Add text to images at the point (x, y)

<svg width="400" height="50"> <text x="20" y="40" font-size="25" > Let it snow, let it snow, let it snow! </text> </svg>

Try including these other attributes with text:

  • Stretch text to a length: textLength="300"
  • Change how text stretches (values: spacing, spacingAndGlyphs): lengthAdjust="spacing"
  • Rotate the text: rotate="10"
<svg width="400" height="100"> <text x="0" y="20" font-size="25" textLength="300" lengthAdjust="spacing"> Let it snow! </text> <text x="0" y="60" font-size="25" textLength="300" lengthAdjust="spacingAndGlyphs"> Let it snow! </text> <text x="20" y="100" font-size="25" textLength="300" lengthAdjust="spacing" rotate="-90"> Let it snow! </text> </svg>

Try using the other attributes you've learned.

<svg width="400" height="50"> <text x="20" y="40" font-size="45" textLength="300" rotate="5" fill="transparent" stroke="plum" stroke-width="2"> Let it snow! </text> </svg>

Transform

Transforms modify the position of items they are applied to. The three transforms are translate, scale and rotate.

Translate

Translate moves the item to a new (x, y) point. If a rectangle is positioned at (5, 10) and a translation of (2, 3) is applied to it then the rectangle's new position is (7, 13).

  • Move item to a new point: transform="translate(10 10)"
<svg width="400" height="100"> <rect x="100" y="30" width="60" height="40" fill="pink" /> <rect x="100" y="30" width="60" height="40" fill="mediumvioletred" fill-opacity=".8" transform="translate(10 10)" /> </svg>

Scale

Scale changes the size of the item. The size can be changed in both the x and y direction. When a scale is applied to an item, for example a rectangle, then it is not only applied to the width and height of the rectangle but also to its (x, y) position.

  • Make an image twice as big: transform="scale(2 2)"
<svg width="400" height="100"> <rect x="100" y="30" width="60" height="40" fill="pink" /> <rect x="55" y="20" width="60" height="40" fill="mediumvioletred" fill-opacity=".8" transform="scale(2 2)" /> </svg>

Notice that the (x, y) position of the rectangle the transform was applied to has been halved. This is because the scale doubled these values as well, which results in the actual (x, y) value of (110, 40).

This can be avoided if we use translate to position the item, the scale transform does not impact the translate transform like it does the (x, y) position.

  • Position item with translate and scale it: transform="translate(110 40) scale(2 2)"
<svg width="400" height="100"> <rect x="100" y="30" width="60" height="40" fill="pink" /> <rect x="0" y="0" width="60" height="40" fill="mediumvioletred" fill-opacity=".8" transform="translate(110 40) scale(2 2)" /> </svg>

Rotate

Rotate an item a number of degrees at a point (x, y).

  • Rotate 45 degrees at the point (100, 30): transform="rotate(45 100 30)"
<svg width="400" height="100"> <rect x="100" y="30" width="60" height="40" fill="pink" /> <rect x="100" y="30" width="60" height="40" fill="mediumvioletred" fill-opacity=".8" transform="rotate(45 100 30)" /> </svg>

Try applying transforms to different shapes: circle, ellipse and text. Include other attributes you have learned while you experiment.

Animation

Animate an attribute

Changes an attribute of an item over a length of time. For example, the width of a rectangle can grow from a smaller size to a larger size. We use the animate within an item and the attributeName attribute to say what attribute of the item to animate. We will tell the animation the beginning value, from, the ending value, to, and the number of seconds, dur, for it to take.

  • Attribute to animate: attributeName="width"
  • Begin: from="100"
  • End: to="150"
  • How long: dur="3s"
  • How many times to perform the animation: repeatCount="indefinite"
<svg width="400" height="100"> <rect x="100" y="10" width="100" height="50" fill="aliceblue" > <animate attributeName="width" from="100" to="150" dur="3s" repeatCount="indefinite" /> </rect> </svg>

Multiple attributes can be changed on a single item.

  • Animate width: attributeName="width" from="150" to="0"
  • Animate height: attributeName="height" from="0" to="50"
<svg width="400" height="75"> <rect x="100" y="10" width="100" height="50" fill="aliceblue" > <animate attributeName="width" from="150" to="0" dur="3s" repeatCount="indefinite" /> <animate attributeName="height" from="0" to="50" dur="3s" repeatCount="indefinite" /> </rect> </svg>

Try animating the font-size, rotate and stroke-width attributes of a text item. Try animating other items and attributes.

Animation transforms

Translate

Move an item from one location to another and have the movement occur over a length of time. The type of movement we will perform is called translate. Similarly to the animate element we will tell the animation what point to begin, from, what point to end, to, and the number of seconds, dur, for it to take.

  • The type of animation: type="translate"
  • The beginning (x, y) point: from="50 0"
  • The ending (x, y) point: to="200 0"
  • How long: dur="10s"
  • How many times to perform the animation: repeatCount="indefinite"
<svg width="400" height="50"> <circle cx="50" cy="30" r="20" fill="yellow"> <animateTransform attributeName="transform" type="translate" from="0 0" to="200 0" dur="10s" repeatCount="indefinite" /> </circle> </svg>

Rotate

Rotate an item from a starting degree at an (x, y) point to an ending degree at another (x, y) point.

  • The type of animation: type="rotate"
  • The beginning degree and (x, y) point: from="50 0"
  • The ending degree and (x, y) point: to="200 0"
<svg width="400" height="100"> <rect x="100" y="30" width="60" height="40" fill="indigo"> <animateTransform attributeName="transform" type="rotate" dur="10s" from="0 130 50" to="360 130 50" repeatCount="indefinite" /> </rect> </svg>

Scale

changes the size of an item. Once again, when an item is scaled the item's position will also change. Therefore we once again use the translate transform to position the item. For the animation there is one additional attribute which needs to be set. The additive attribute will be set to sum, so that the animation does not override the position set.

  • The type of animation: type="scale"
  • The beginning size in the x and y direction: from="50 0"
  • The ending size in the x and y direction: to="200 0"
  • Use the translate position with the animation: additive="sum"
<svg width="400" height="100"> <circle cx="0" cy="0" r="20" fill="deeppink" transform="translate(100 50)"> <animateTransform attributeName="transform" type="scale" additive="sum" dur="3s" from=".5 .5" to="2 2" repeatCount="indefinite" /> </circle> </svg>

Try combining these animations with other items. Try creating multiple items each with their own animation.

Polygon

Custom shapes can be created by specifying a series of (x, y) points. The last point will automatically connect to the first point.

<svg width="400" height="100"> <polygon points="100,30 200,100 300,60 200,10" fill="orange" /> </svg>

There is no practical limit to the number of points you can specify. You are free to make as complex image as you like.

<svg width="400" height="100"> <polygon points="100,50 130,100 180,60 220,80 300,50 220,20 180,40 130,0" fill="paleturquoise" /> </svg>

Try creating your own shape.

Other attributes can be used with custom shapes. In this example transform is used to translate and rotate the shape.

<svg width="400" height="200"> <polygon points="0,50 30,100 80,60 120,80 200,50 120,20 80,40 30,0" fill="paleturquoise" transform="translate(200 0) rotate(90)" /> </svg>

Try applying other attributes to your own shapes.

Gradients

Linear

Gradients transition an item from one color to another color. Linear gradients transition the color in a straight line, from left to right by default. The first offset is used to specify what percent along the straight line the color should begin the transition. The next offset is when the next color will end the transition.

The gradient information is specified by itself in a defs section, definition section. Each definition has an id attribute which then can be used on an item to apply the gradient to that item.

  • The id of the gradient: linearGradient id="NoBlur"
  • The first offset begins transitioning away from the color darkblue immediately: stop offset="0%" stop-color="darkblue"
  • The next offset ends the color transition, resulting in the color only becoming deepskyblue at the very end of the item: stop offset="100%" stop-color="deepskyblue"
  • The gradient to apply to the item: fill="url(#CompleteGradient)"
<svg width="250" height="120"> <defs> <linearGradient id="CompleteGradient"> <stop offset="0%" stop-color="darkblue"/> <stop offset="100%" stop-color="deepskyblue"/> </linearGradient> </defs> <rect x="10" y="10" width="150" height="100" fill="url(#CompleteGradient)"/> </svg>

Any number of offsets and colors can be specified.

<svg width="250" height="120"> <defs> <linearGradient id="MultiGradientOffsets"> <stop offset="0%" stop-color="darkblue"/> <stop offset="50%" stop-color="dodgerblue"/> <stop offset="100%" stop-color="deepskyblue"/> </linearGradient> </defs> <rect x="10" y="10" width="150" height="100" fill="url(#MultiGradientOffsets)"/> </svg>

Try different offset values and including additional offsets.

The direction of the gradient can be specified, so it is applied in a different direction than left to right. The direction can be specified in both an x and y direction using values from 0 to 1, where 1 means it will be applied to the item completely in that direction.

  • Apply gradient from upper left to bottom right: x1="0" y1="0" x2="1" y2="1"
  • Apply gradient from bottom left to the right but only halfway to the top: x1="0" y1="1" x2="1" y2=".5"
<svg width="400" height="100"> <defs> <linearGradient id="CornerBlur" x1="0" y1="0" x2="1" y2="1"> <stop offset="0%" stop-color="darkblue"/> <stop offset="100%" stop-color="deepskyblue"/> </linearGradient> <linearGradient id="CornerBlurReverse" x1="0" y1="1" x2="1" y2=".5"> <stop offset="0%" stop-color="deepskyblue"/> <stop offset="100%" stop-color="darkblue"/> </linearGradient> </defs> <rect x="0" y="10" width="200" height="100" fill="url(#CornerBlur)"/> <rect x="200" y="10" width="200" height="100" fill="url(#CornerBlurReverse)"/> </svg>

Try adjusting the direction of the gradient on items.

Radial

Radial gradients transition from one color to another starting at a point and moving outwards in all directions. Radial gradients are specified the same way as linear gradients.

An opacity, making the color see through, can also be used with a gradient.

  • Make it see through (values: 0.0-1.0): stop-opacity=".8"
<svg width="400" height="100"> <defs> <radialGradient id="RadialGradient1"> <stop offset="0%" stop-color="orchid"/> <stop offset="100%" stop-color="mediumorchid"/> </radialGradient> <radialGradient id="RadialGradient2"> <stop offset="0%" stop-color="violet"/> <stop offset="50%" stop-color="orchid" /> <stop offset="100%" stop-color="mediumorchid"/> </radialGradient> <radialGradient id="RadialGradient3"> <stop offset="0%" stop-color="violet" stop-opacity=".8"/> <stop offset="50%" stop-color="orchid" stop-opacity=".9"/> <stop offset="100%" stop-color="mediumorchid" stop-opacity=".95"/> </radialGradient> </defs> <circle cx="50" cy="50" r="40" fill="url(#RadialGradient1)" /> <circle cx="150" cy="50" r="40" fill="url(#RadialGradient2)" /> <circle cx="250" cy="50" r="40" fill="url(#RadialGradient3)" /> </svg>

Try combining gradients with other items you have learned. Here are two examples which include a gradient to an image we've already seen.

<svg width="400" height="150"> <defs> <radialGradient id="AnimatedGradient"> <stop offset="1%" stop-color="hotpink"/> <stop offset="99%" stop-color="deeppink"/> </radialGradient> </defs> <circle fill="url(#AnimatedGradient)" cx="0" cy="0" r="30" transform="translate(100 75)"> <animateTransform attributeName="transform" type="scale" additive="sum" dur="3s" from=".5 .5" to="2 2" repeatCount="indefinite" /> </circle> </svg>
<svg width="400" height="200"> <defs> <radialGradient id="PolygonGradient" r=".7" cx=".2" > <stop offset="1%" stop-color="paleturquoise"/> <stop offset="99%" stop-color="deepskyblue"/> </radialGradient> </defs> <polygon points="0,50 30,100 80,60 120,80 200,50 120,20 80,40 30,0" fill="url(#PolygonGradient)" transform="translate(200 0) rotate(90)" /> </svg>

Filters

Filters provide a way to apply an effect to an image. There are many kinds of filters. This is the list of supported filters. We will only be using a single filter, feGaussianBlur, which blurs an item.

  • How much to blur the image: stdDeviation="5 2"
    • If one value is supplied then both the x and y directions are blurred the same, else the first value will blur the x direction and the second value the y direction
<svg width="400" height="100"> <defs> <filter id="SameBlur"> <feGaussianBlur stdDeviation="5" /> </filter> <filter id="DifferingBlur"> <feGaussianBlur stdDeviation="5 0" /> </filter> </defs> <circle cx="50" cy="50" r="40" fill="springgreen"/> <circle cx="150" cy="50" r="40" fill="springgreen" filter="url(#SameBlur)"/> <circle cx="250" cy="50" r="40" fill="springgreen" filter="url(#DifferingBlur)"/> </svg>

Try applying this filter to an item you create. Adjust the stdDeviation value to see how it effects your item.

Groups and Use

Several items can be created within a group. This allows you to treat the entire group as a single item.

In this example four circles are created. Then as a group they are moved to another location and rotated

<svg width="400" height="75"> <g fill="gold" transform="translate(40 0) rotate(25)"> <circle cx="20" cy="15" r="8" /> <circle cx="37" cy="15" r="6" /> <circle cx="50" cy="15" r="4" /> <circle cx="60" cy="15" r="2" /> </g> </svg>

A group can also be defined and then used elsewhere. The group can also be used multiple times.

In this example a group of circles is defined. Then the group is used twice, but they are placed at different positions and are rotated differently.

  • Define a group: g id="DotsTransformed"
  • Use the defined group: use href="#DotsTransformed"
<svg width="400" height="50"> <def> <g id="DotsTransformed" fill="gold"> <circle r="8" /> <circle r="6" transform="translate(17 0)" /> <circle r="4" transform="translate(30 0)" /> <circle r="2" transform="translate(40 0)" /> </g> </def> <use href="#DotsTransformed" transform="translate(50 20) rotate(20)"/> <use href="#DotsTransformed" transform="translate(140 20) rotate(160)"/> </svg>

Try defining a group of items. Then use it multiple times. Make each usage a little different than other usages. For example what happens if you apply a gradient to the group or animate the group?

Viewbox

The viewBox attribute allows you to change how the image is viewed. We'll first look at the last two values of the attribute. The last two values are the width and height of the view. If these values are the same as the image then the image is not changed. If the values are smaller than the image then, in effect, the image becomes larger, the image is zoomed in. If the values are larger then the image becomes smaller, the image is zoomed out.

In this example the view of the first image has not been changed. In the second the image is zoomed in. And in the third the image is zoomed out.

<svg width="150" height="150" viewBox="0 0 150 150"> <rect width="100%" height="100%" fill="lightsteelblue"/> <rect x="5" y="5" rx="5" ry="5" width="75" height="75" fill="springgreen"/> <circle cx="100" cy="100" r="25" fill="greenyellow" /> </svg> <svg width="150" height="150" viewBox="0 0 100 100" > <rect width="100%" height="100%" fill="lightsteelblue"/> <rect x="5" y="5" rx="5" ry="5" width="75" height="75" fill="springgreen"/> <circle cx="100" cy="100" r="25" fill="greenyellow" /> </svg> <svg width="150" height="150" viewBox="0 0 200 200" > <rect width="100%" height="100%" fill="lightsteelblue"/> <rect x="5" y="5" rx="5" ry="5" width="75" height="75" fill="springgreen"/> <circle cx="100" cy="100" r="25" fill="greenyellow" /> </svg>

Now let's go back and look at the first two values. These values make up the (x, y) point to begin viewing the image at. For example, if an image has a width of 100 and height of 100 then if a viewBox of 10 10 is used the view of the image will be shifted both in the x and y positions 10 spots into the image.

This example is the same as the one above except an initial viewing point has been specified for each image, which shifts the view of the image to the right and down.

<svg width="150" height="150" viewBox="40 40 150 150"> <rect width="150%" height="150%" fill="lightsteelblue"/> <rect x="5" y="5" rx="5" ry="5" width="75" height="75" fill="springgreen"/> <circle cx="100" cy="100" r="25" fill="greenyellow" /> </svg> <svg width="150" height="150" viewBox="30 30 100 100" > <rect width="150%" height="150%" fill="lightsteelblue"/> <rect x="5" y="5" rx="5" ry="5" width="75" height="75" fill="springgreen" /> <circle cx="100" cy="100" r="25" fill="greenyellow" /> </svg> <svg width="150" height="150" viewBox="50 50 200 200" > <rect width="150%" height="150%" fill="lightsteelblue"/> <rect x="5" y="5" rx="5" ry="5" width="75" height="75" fill="springgreen"/> <circle cx="100" cy="100" r="25" fill="greenyellow" /> </svg>

Try different viewBox widths and heights. Also try different starting points for viewing the image.
