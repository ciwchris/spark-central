Transform

Transforms modify the position of items they are applied to. The three transforms are translate, scale, rotate and skew.

Translate

Translate moves the item to a new (x, y) point. If a rectangle is positioned at (5, 10) and a translation of (2, 3) is applied to it then the rectangle's new position is (7, 13).

  • Movie item to new point: transform="translate(10 10)"
<svg width="400" height="100"> <rect x="100" y="30" width="60" height="40" fill="pink" /> <rect x="100" y="30" width="60" height="40" fill="mediumvioletred" fill-opacity=".8" transform="translate(10 10)" /> </svg>

Scale

Scale changes the size of the item. The size can be changed in both the x and y direction. When a scale is applied to an item, for example a rectangle, then it is not only applied to the width and height of the rectangle but also to its (x, y) position.

  • Make image twice as big: transform="scale(2 2)"
<svg width="400" height="100"> <rect x="100" y="30" width="60" height="40" fill="pink" /> <rect x="55" y="20" width="60" height="40" fill="mediumvioletred" fill-opacity=".8" transform="scale(2 2)" /> </svg>

Notice that the (x, y) position of the rectangle the transform was applied to has half the expected values. This is because the scale double these values as well, which results in the actual (x, y) values of (110, 40).

This can be avoided if we use translate to position the item, the scale transform does not impact the translate transform like it does the (x, y) position of the time.

  • Position item with translate and scale it: transform="translate(110 40) scale(2 2)"
<svg width="400" height="100"> <rect x="100" y="30" width="60" height="40" fill="pink" /> <rect x="0" y="0" width="60" height="40" fill="mediumvioletred" fill-opacity=".8" transform="translate(110 40) scale(2 2)" /> </svg>

Rotate

Rotates an item a number of degrees at a point (x, y).

  • Rotate 45 degrees at the point (100, 30): transform="rotate(45 100 30)"
<svg width="400" height="100"> <rect x="100" y="30" width="60" height="40" fill="pink" /> <rect x="100" y="30" width="60" height="40" fill="mediumvioletred" fill-opacity=".8" transform="rotate(45 100 30)" /> </svg>

Skew

Skew pulls an image in a direction. For a rectangle it will pull the bottom right corner in either the x or y direction.

  • Skew in the x direction: transform="translate(20 20) skewX(10)"
  • Skew in the y direction: transform="translate(110 20) skewY(10)"
  • Skew in both the x and y directions: transform="translate(210 20) skewX(10) skewY(10)"
<svg width="400" height="100"> <rect x="10" y="10" width="60" height="40" fill="pink" /> <rect x="0" y="0" width="60" height="40" fill="mediumvioletred" fill-opacity=".8" transform="translate(20 20) skewX(10)" /> <rect x="100" y="10" width="60" height="40" fill="pink" /> <rect x="0" y="0" width="60" height="40" fill="mediumvioletred" fill-opacity=".8" transform="translate(110 20) skewY(10)" /> <rect x="200" y="10" width="60" height="40" fill="pink" /> <rect x="0" y="0" width="60" height="40" fill="mediumvioletred" fill-opacity=".8" transform="translate(210 20) skewX(10) skewY(10)" /> </svg>

Animation

Animate an attribute

Changes an attribute of an item over a length of time. For example, the width of a rectangle can grow from a smaller size to a larger size. We use the animate within an item and the attributeName attribute to tell what attribute of the item to animate. We will tell the animation the beginning value, from, the ending value, to, and the number of seconds, dur, for it to take.

  • Attribute to animate: attributeName="width"
  • Begin: from="100"
  • End: to="150"
  • How long: dur="3s"
  • How many times to perform the animation: repeatCount="indefinite"
<svg width="400" height="100"> <rect x="100" y="10" width="100" height="50" fill="aliceblue" > <animate attributeName="width" from="100" to="150" dur="3s" repeatCount="indefinite" /> </rect> </svg>

Multiple attributes can be changed on a single item.

  • Animate width: attributeName="width" from="150" to="0"
  • Animate height: attributeName="height" from="0" to="50"
<svg width="400" height="75"> <rect x="100" y="10" width="100" height="50" fill="aliceblue" > <animate attributeName="width" from="150" to="0" dur="3s" repeatCount="indefinite" /> <animate attributeName="height" from="0" to="50" dur="3s" repeatCount="indefinite" /> </rect> </svg>

Try animating the font-size, rotate and stroke-width attributes of a text item. Try animating other items and attributes.

Animation transforms

Translate

Move an item from one location to another and have the movement occur over a length of time. The type of movement we will perform is called translate. Similarly to the animate element we will tell the animation what point to begin, from, what point to end, to, and the number of seconds, dur, for it to take.

  • The type of animation: type="translate"
  • The beginning (x, y) point: from="50 0"
  • The ending (x, y) point: to="200 0"
  • How long: dur="10s"
  • How many times to perform the animation: repeatCount="indefinite"
<svg width="400" height="50"> <circle cx="50" cy="30" r="20" fill="yellow"> <animateTransform attributeName="transform" type="translate" from="50 0" to="200 0" dur="10s" repeatCount="indefinite" /> </circle> </svg>

Rotate

Rotates an item from a starting degree at an (x, y) point to an ending degree to another (x, y) point.

  • The type of animation: type="rotate"
  • The beginning degree and (x, y) point: from="50 0"
  • The ending degree and (x, y) point: to="200 0"
<svg width="400" height="100"> <rect x="100" y="30" width="60" height="40" fill="indigo"> <animateTransform attributeName="transform" type="rotate" dur="10s" from="0 130 50" to="360 130 50" repeatCount="indefinite" /> </rect> </svg>

Scale

Changes the size of an item. Once again, when an item is scaled the items position will also change. Therefore we once again use the translate transform to position the item. For the animation there is one additional attribute which needs to be set. The additive attribute will be set to sum, so that the animation does not override the position set.

  • The type of animation: type="scale"
  • The beginning size in the x and y direction: from="50 0"
  • The ending size in the x and y direction: to="200 0"
  • Use the translate position with the animation: additive="sum"
<svg width="400" height="100"> <circle cx="0" cy="0" r="20" fill="deeppink" transform="translate(100 50)"> <animateTransform attributeName="transform" type="scale" additive="sum" dur="3s" from=".5 .5" to="2 2" repeatCount="indefinite" /> </circle> </svg>

Gradients

Linear

Gradients transition an item from one color to another color. Linear gradients transition the color in a straight line from left to right by default. The first offset is used to specify what percent along the straight line the color should begin the transition. The next offset is when the next color will end the transition.

The gradient information is specified by itself in a defs section, definition section. Each definition has an id attribute which then can be used on an item to apply the gradient to that item.

  • The id of the gradient: linearGradient id="NoBlur"
  • The first offset begins transitioning away from the color darkblue immediately: stop offset="0%" stop-color="darkblue"
  • The next offset ends the color transition, resulting in the color only becoming deepskyblue at the very end of the item: stop offset="100%" stop-color="deepskyblue"
  • The gradient to apply to the item: fill="url(#CompleteGradient)"
<svg width="250" height="120"> <defs> <linearGradient id="CompleteGradient"> <stop offset="0%" stop-color="darkblue"/> <stop offset="100%" stop-color="deepskyblue"/> </linearGradient> </defs> <rect x="10" y="10" width="150" height="100" fill="url(#CompleteGradient)"/> </svg>

Any number of offsets and colors can be specified.

<svg width="250" height="120"> <defs> <linearGradient id="MultiGradientOffsets"> <stop offset="0%" stop-color="darkblue"/> <stop offset="50%" stop-color="dodgerblue"/> <stop offset="100%" stop-color="deepskyblue"/> </linearGradient> </defs> <rect x="10" y="10" width="150" height="100" fill="url(#MultiGradientOffsets)"/> </svg>

Try different offset values and including additional offsets.

The direction of the gradient can be specified, so it is not applied in a different direction than left to right. The direction can be specified in an x and y direction using values from 0 to 1, where 1 means it will be applied to the item completely in that direction.

  • Apply gradient from upper left to bottom right: x1="0" y1="0" x2="1" y2="1"
  • Apply gradient from bottom left to the right but only halfway to the top: x1="0" y1="1" x2="1" y2=".5"
<svg width="400" height="100"> <defs> <linearGradient id="CornerBlur" x1="0" y1="0" x2="1" y2="1"> <stop offset="0%" stop-color="darkblue"/> <stop offset="100%" stop-color="deepskyblue"/> </linearGradient> <linearGradient id="CornerBlurReverse" x1="0" y1="1" x2="1" y2=".5"> <stop offset="0%" stop-color="deepskyblue"/> <stop offset="100%" stop-color="darkblue"/> </linearGradient> </defs> <rect x="0" y="10" width="200" height="100" fill="url(#CornerBlur)"/> <rect x="200" y="10" width="200" height="100" fill="url(#CornerBlurReverse)"/> </svg>

Try adjusting the direction of the gradient on items.

Radial

Radial gradients transition from one color to another starting at a point and moving outwards in all directions. Radial gradients are specified the same way as linear gradients.

An opacity, making the color see through, can also be used with a gradient.

  • Make it see through (values: 0.0-1.0): stop-opacity=".8"
<svg width="400" height="100"> <defs> <radialGradient id="RadialGradient1"> <stop offset="0%" stop-color="orchid"/> <stop offset="100%" stop-color="mediumorchid"/> </radialGradient> <radialGradient id="RadialGradient2"> <stop offset="0%" stop-color="violet"/> <stop offset="50%" stop-color="orchid" /> <stop offset="100%" stop-color="mediumorchid"/> </radialGradient> <radialGradient id="RadialGradient3"> <stop offset="0%" stop-color="violet" stop-opacity=".8"/> <stop offset="50%" stop-color="orchid" stop-opacity=".9"/> <stop offset="100%" stop-color="mediumorchid" stop-opacity=".95"/> </radialGradient> </defs> <circle cx="50" cy="50" r="40" fill="url(#RadialGradient1)" /> <circle cx="150" cy="50" r="40" fill="url(#RadialGradient2)" /> <circle cx="250" cy="50" r="40" fill="url(#RadialGradient3)" /> </svg>

How far the gradient extends as well as the location of the gradient and its focal point can be adjusted.

  • The center: cx=".3" cy=".7"
  • The focal point: fx=".35" fy=".6"

In these examples radial gradients are positioned within rectangles. The first specifies how far the gradient extends. The second sets the center of the gradient. The third adjusts where the gradient is focused.

  • How much to blur the image. If one value is supplied to blur both the x and y directions the same, else use first value for the x direction and the second value for the y direction: stdDeviation="5 2"

Viewbox

The viewBox attribute allows you to change how the image is viewed. The first two values is the (x, y) point to begin viewing the image at. For example, if an image has a width of 100 and height of 100 then if a viewBox of 10 10 is used then the view of the image will be shifted both in the x and y positions 10 points into the image.

The next two values are the width and height of the view. If these values are the same as the image then the image is not changed. If the values are smaller than the image then in effect the image becomes larger, the image is zoomed in. If the values are larger then the image becomes smaller, the image is zoomed out.

In this example the view of the first image has not been change. In the second the image is zoomed in. And in the third the image is zoomed out.

<svg width="150" height="150" viewBox="0 0 150 150"> <rect width="200%" height="200%" fill="lightsteelblue"/> <rect x="5" y="5" rx="5" ry="5" width="75" height="75" fill="springgreen"/> <circle cx="100" cy="100" r="25" fill="greenyellow" /> </svg> <svg width="150" height="150" viewBox="0 0 100 100" > <rect width="200%" height="200%" fill="lightsteelblue"/> <rect x="5" y="5" rx="5" ry="5" width="75" height="75" fill="springgreen"/> <circle cx="100" cy="100" r="25" fill="greenyellow" /> </svg> <svg width="150" height="150" viewBox="0 0 200 200" > <rect width="200%" height="200%" fill="lightsteelblue"/> <rect x="5" y="5" rx="5" ry="5" width="75" height="75" fill="springgreen"/> <circle cx="100" cy="100" r="25" fill="greenyellow" /> </svg>

Paths and Archs

Paths and archs are a more challenging topic. They allow for the creation of complex images. Below are some of the basics and show how they work. When working with paths and arks most of the time another tool will be used which generates the image for you and outputs the SVG information.

This first example shows you a square can be created using paths. The starting point is specified at an (x, y) position using M10 10, the absolute (x, y) point. Next h 80 is used to draw a horizontal line to the right, relative to the previous point. v 80 does this same thing, except it draws a vertical line down. h -80 draws a line from right to left. Finally Z completes the rectangle by automatically connecting the last point to the first point.

<svg width="200" height="100"> <path d="M10 10 h 80 v 80 h -80 Z" fill="transparent" stroke="royalblue" stroke-width="10"/> </svg>

Archs are created using seven separate values.

  • rx: radius in the x direction: M10 10 a 1 0 0 0 0 0 0
  • ry: radius in the y direction: ryM10 10 a 1 1 0 0 0 0 0
  • x-axis-rotation: rotation of the arc: M10 10 a 1 1 45 0 0 0 0
  • large-arc-flag: begin moving at positive or negative anglesM10 10 a 1 1 45 1 0 0 0
  • sweep-flag: direction the arc will travel around a given circle: M10 10 a 1 1 45 1 1 0 0
  • dx: ending x location: M10 10 a 1 1 45 1 1 100 0
  • dy: ending y location: M10 10 a 1 1 45 1 1 100 10

Specifying rx and ry, values 1 and 1. And ending point of 100, dx and 0, dy are used.

<svg width="400" height="150"> <path d="M10 10 a 1 1 0 0 0 100 0" fill="cadetblue" stroke="cadetblue" /> <path d="M120 10 a 5 1 0 0 0 100 0" fill="lightseagreen" stroke="lightseagreen" /> <path d="M240 10 a 1 2 0 0 0 100 0" fill="mediumseagreen" stroke="mediumseagreen" /> </svg>

Same as the image above but with an ending y position, dy, set.

<svg width="400" height="150"> <path d="M10 10 a 1 1 0 0 0 100 20" fill="fuchsia" stroke="fuchsia" /> <path d="M150 10 a 5 1 0 0 0 100 20" fill="deeppink" stroke="deeppink" /> <path d="M270 10 a 1 2 0 0 0 100 20" fill="mediumvioletred" stroke="mediumvioletred" /> </svg>

Rotates the arch, x-axis-rotation, at an increasing angle, i.e. number of degrees.

<svg width="650" height="110"> <path d="M30 0 a 2 1 0 0 0 100 0" fill="coral" stroke="coral" /> <path d="M150 0 a 2 1 22 0 0 100 0" fill="orange" stroke="orange" /> <path d="M280 0 a 2 1 45 0 0 100 0" fill="darkorange" stroke="darkorange" /> <path d="M410 0 a 2 1 57 0 0 100 0" fill="tomato" stroke="tomato" /> <path d="M540 0 a 2 1 90 0 0 100 0" fill="orangered" stroke="orangered" /> </svg>
